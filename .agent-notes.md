### Lições aprendidas

1. **Snapshot antes de edições:** Sempre usar `git stash push -m "snapshot: before <change>"` antes de edições parciais com `edit_file`
2. **Handlers globais para onclick:** Quando botões HTML usam `onclick` inline, os handlers DEVEM ser expostos ao `window` explicitamente ou definidos no escopo correto
3. **Verificar arquivo após restauração:** Sempre verificar o conteúdo do arquivo após `git checkout` para garantir que não há corrupção
4. **Usar npm run release X.Y.Z:** O script automatiza update, sync, build, commit, tag e push. Evitar push manual para garantir consistência.
